# Tech Stack

## Overview

AI-native medical claims review platform on AWS. Ingests scanned claim documents, uses Textract + Bedrock (Claude) to parse/classify/extract structured fields with confidence scoring, auto-routes high-confidence documents and queues low-confidence ones for human review, and closes the loop by feeding reviewer corrections back into an evaluation dataset that drives automated prompt optimization for an assistant agent. Built as an AWS-native mirror of a Databricks reference architecture (Unity Catalog/Delta/Lakeflow/Lakebase/MLflow) for interview preparation.

## Core Technologies

### Language

**Primary**: Python
**Version**: 3.12+
(Frontend uses TypeScript — see Frontend section)

### Framework

**Framework**: FastAPI (backend API), LangGraph (agent orchestration)
**Version**: FastAPI 0.11x, LangGraph 0.2.x
**Rationale**: FastAPI gives async I/O + auto OpenAPI docs with minimal boilerplate; LangGraph gives explicit, inspectable state machines for the agent (matches the "stateful LangGraph agent" pattern from the reference architecture) rather than a black-box agent framework.

### Runtime

**Runtime**: AWS Lambda (Python 3.12 runtime) for pipeline steps; ECS Fargate for long-running backend/agent services
**Version**: Lambda Python3.12 managed runtime; Fargate latest platform version

## Data Layer

### Database

**Type**: Relational (+ vector extension)
**Database**: Amazon Aurora Serverless v2 — PostgreSQL
**Version**: PostgreSQL 16, pgvector extension enabled

### ORM / Data Access

**Tool**: SQLAlchemy 2.x + Alembic for migrations
**Rationale**: Explicit schema control and migration history matter here since Aurora plays the same "system of record + audit trail" role Lakebase plays in the original architecture.

### Caching

Not configured initially. Revisit if the reviewer dashboard or agent memory lookups show latency issues (candidate: ElastiCache Serverless for Redis).

## API Layer

### API Style

**Style**: REST (FastAPI), plus one Server-Sent Events (SSE) endpoint for streaming the assistant agent's responses

### Documentation

**Tool**: FastAPI automatic OpenAPI schema + Swagger UI (`/docs`)
