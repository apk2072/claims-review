# Project Constitution

> Universal policies that apply to ALL code in this project.
> This file is always inherited from root — modules cannot override it.

## Project Nature

- **Solo learning project**: built for hands-on interview preparation (Databricks/AI-engineering interview), not production.
- Optimize for understanding *why* each piece exists over shipping speed — this project's value is the learning, not just the working demo.
- Real AWS costs are acceptable to the owner, but prefer serverless/scale-to-zero services and tear-down scripts to keep spend small.

## Git Workflow

- **Commit Style**: Conventional commits (feat:, fix:, chore:, docs:, refactor:, test:)
- **Branch Strategy**: Feature branch per work item, merged to main when a work item completes
- **Main Branch**: main is always deployable (or at least always `cdk synth`-able)

## Code Review

- Solo project — no external reviewers. Run `/code-review` on each work item's diff before merging as a self-check.
- No secrets or generated credentials committed, even in a personal repo.

## CI/CD

- Not configured initially — deploys are manual (`cdk deploy`) so the owner sees and understands each change.
- GitHub Actions may be added later as a stretch work item once the core platform works.

## Security Policies

- **No secrets in code** — use environment variables and AWS Secrets Manager / SSM Parameter Store.
- **No real PHI ever** — this project only ever processes synthetic/fake claims documents, even though the domain is medical claims. This is a hard rule, not a preference.
- IAM roles follow least-privilege per service (Lambda, ECS task roles, Bedrock/Textract access scoped narrowly).
- Dependencies from trusted sources (PyPI/npm official registries only).

## Documentation

- Each top-level component (`pipeline/`, `backend/`, `frontend/`, `agent/`, `infra/`) has its own README covering purpose and how to run/test it locally.
- `system-architecture.md` kept up to date as components are added — it is the map back to the original Databricks reference architecture this project mirrors.
